## ‚öôÔ∏è How Each Protocol Works

### ‚úÖ TCP (Transmission Control Protocol)
- **Working**:
  - 3-way handshake (SYN, SYN-ACK, ACK)
  - Ensures ordered, reliable data delivery
  - Retransmits lost packets
- **Use Case in Terminal Tools**:
  - Reliable remote control (e.g., `Netcat`, `SSH`)

### ‚úÖ UDP (User Datagram Protocol)
- **Working**:
  - Sends data without handshake
  - No guarantee of delivery or order
  - Lower overhead, faster
- **Use Case in Terminal Tools**:
  - Fast data like video/audio streaming
  - Port scanning (e.g., `nmap -sU`)

### ‚úÖ ICMP (Internet Control Message Protocol)
- **Working**:
  - Not used for data transfer
  - Used for diagnostics (errors, unreachable, TTL expired)
  - Tools like `ping`, `traceroute` use it
- **Use Case in Terminal Tools**:
  - Check if a host is up
  - Path discovery (Traceroute)

### ‚úÖ SMB (Server Message Block)
- **Working**:
  - Operates over TCP (port 445)
  - Allows shared access to files, printers
  - Requires authentication
- **Use Case in Terminal Tools**:
  - Network enumeration
  - File upload/download in internal networks
  - Hacking tools use it (e.g., `EternalBlue`)

## ‚ùå Protocol Limitations

| **Protocol** | **Limitations**                                                              |
|--------------|-------------------------------------------------------------------------------|
| TCP          | Slower due to handshaking, not good for real-time use                        |
| UDP          | Unreliable, no retransmission or ordering                                    |
| ICMP         | Often blocked by firewalls, limited to diagnostics                           |
| SMB          | Vulnerable to exploits (e.g., SMBv1), requires authentication, heavy usage   |
